# R3.M9 Release Baseline Acceptance

Status: accepted for R3 finalization
Date: 2026-04-24
Scope: `tests/perf/gate_pr.ps1`, `tests/perf/gate_nightly.ps1`

## Summary

R3 changed the runtime and compiler surface substantially.

The perf gates now show baseline drift against the older R2/R3-start baseline.

The important release signal is:

- hard budgets are still within limits;
- failures are baseline-comparison failures;
- no benchmark currently shows an absolute budget failure in the R3.M9 quick report.

## Accepted Reason

This override accepts the R3.M9 baseline drift so the checklist can close with a documented decision.

It does not remove performance tracking.

It moves the next optimization/rebaseline work into R4.

## Follow-Up

R4 must create a fresh baseline after release stabilization.

Track this as:

- `R4.M0` stabilization base;
- perf baseline refresh for `windows-AMD64`;
- follow-up investigation for startup/binary-size drift.

## Guardrail

This file is only valid for the R3.M9 finalization window.

If any benchmark starts failing absolute hard budgets, this override must not be used.
