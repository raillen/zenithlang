# Cognição Visual e Zenith

O Zenith não é apenas uma linguagem; é uma ferramenta de suporte cognitivo.

## Por que Zenith é diferente?

### 1. Verticalidade vs Horizontalidade
Muitas linguagens como JS ou C# tendem a crescer para a direita (muitos parênteses, chaves, encadeamentos). O Zenith é desenhado para crescer para baixo. 
- **Benefício**: Facilita o rastreamento ocular, essencial para quem tem dislexia.

### 2. Palavras-Chave em vez de Símbolos
Usamos `and`, `or`, `not`, `is` em vez de `&&`, `||`, `!`, `===`. 
- **Benefício**: Reduz a carga mental de "traduzir" símbolos para conceitos. O cérebro processa palavras familiares de forma mais rápida e automática.

### 3. Blocos Identificados
O uso de `end` para fechar blocos, aliado a uma identação clara, cria "caixas visuais" que o cérebro consegue isolar.
- **Benefício**: Ajuda no foco (TDAH) ao isolar a lógica atual do resto do arquivo.

### 4. Reatividade Explícita
Com `state` e `watch`, você não precisa imaginar quando um valor muda. O código diz exatamente o que ele está observando.
- **Benefício**: Elimina o "rastreamento mental" de variáveis ocultas.

## Dicas de Interface
Para uma experiência completa, recomendamos usar o Zenith com:
- **Fontes Mono-espaçadas**: Como *JetBrains Mono* ou *OpenDyslexic*.
- **Cores Sóbrias**: Temas com baixo contraste de luz (Dark Mode suave).
- **Zenith IDE (Retina)**: Nossa IDE oficial desenhada com estas mesmas regras.
