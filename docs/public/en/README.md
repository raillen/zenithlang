# PublicDocs

> Documentation for the website and language use.
> Audience: user
> Surface: public

## Objective

Teach Zenith with low friction.

Use this folder to:

- first steps;
- installation;
- tutorials;
- small examples;
- tool usage guides;
- material that can go on the website.

Do not use this folder for:

- roadmap;
- checklist;
- report;
- audit;
- internal plans;
- features not yet implemented as if they were current.

## Sections

- `learn/`: didactic trail in order, to learn the language.
-`cookbook/`: Practical answers to common problems.
-`history-and-manifesto.md`: history, motivation and public philosophy of the project.
-`guides/`: existing practical guides.
-`tools/`: Ecosystem tools such as ZPM.
-`licensing/`: license, trademark and contribution.
-`get-started/`: first steps.
-`language/`: language teaching guides.
-`stdlib/`: stdlib usage guides.
-`packages/`: official package guides.

## Rule for user content

Every published page must respond quickly:

- what is it;
- when to use;
- how to write;
- common error;
- current limit.

Use`docs/internal/standards/user-doc-template.md`to maintain this standard.
