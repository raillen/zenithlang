# History and Manifesto

> Audience: user
> Status: current
> Surface: public
> Source of truth: no

## What does Zenith and

Zenith is a language created as a practical study.

It was born to investigate, build and test ideas about:

- artificial intelligence;
- operating systems;
- compilers;
- runtime;
- language design;
- programming ergonomics.

It is also a personal laboratory: a place to learn by doing.

## What Zenith doesn't try to be

Zenith was not created to compete with languages ​​on the market.

It does not attempt to replace Rust, C, C#, Python, JavaScript, Go or any other established language.

It should also not be read as a commercial promise, business product or production-ready platform.

The main objective is study, technical evolution and clarity of thought.

## Why create a language

Creating a language requires understanding several layers at the same time:

- as a syntax and read by humans;
- how a parser understands this syntax;
- how types help avoid errors;
- how errors appear to those who program;
- how a stdlib defines the actual usage experience;
- how a runtime executes language decisions;
- how tools, LSP, docs and packages make the language usable.

Zenith exists because this path teaches more than just reading about these subjects.

## The philosophy of language

Zenith tries to prioritize:

- explicit code;
- low ambiguity;
- readable error messages;
- short and direct documentation;
- small examples;
- clear separation between absence and failure;
- visible mutation;
- packages with predictable contracts.

The language must be friendly to read.

This is especially important for people with ADHD, dyslexia, or difficulty maintaining context in long files.

## AI as part of the study

Zenith is also a study on how AI can help in the development of a language.

This includes:

- write and review documentation;
- create tests;
- compare design decisions;
- detect inconsistencies;
- evolve tools;
- accelerate implementation cycles.

AI does not remove technical responsibility. It increases the need for clear specifications, good testing and honest review.

## Maturity

Zenith should still be treated as an evolving language.

Some parts are experimental.

Some APIs may change.

The repository must make it clear what it is:

- current;
- experimental;
- planned;
- history;
- normative for implementation.

This separation protects the language from confusion and contradictory documentation.

## Invitation

Zenith is, first and foremost, a deep learning project.

If it is useful for other people to study compilers, runtimes, systems, AI or language design, that is already a victory.

The focus is not on beating the market.

The focus is to learn rigorously, build clearly and record decisions honestly.
