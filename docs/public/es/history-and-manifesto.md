## ¿Qué significa Zenith y

Zenith es un lenguaje creado como estudio práctico.

Nació para investigar, construir y probar ideas sobre:

- inteligencia artificial;
- sistemas operativos;
- compiladores;
- tiempo de ejecución;
- diseño de lenguaje;
- ergonomía de programación.

También es un laboratorio personal: un lugar para aprender haciendo.

## Lo que Zenith no intenta ser

Zenith no fue creado para competir con los idiomas del mercado.

No intenta reemplazar Rust, C, C#, Python, JavaScript, Go o cualquier otro lenguaje establecido.

Tampoco debe leerse como una promesa comercial, un producto comercial o una plataforma lista para producción.

El objetivo principal es el estudio, la evolución técnica y la claridad de pensamiento.

## ¿Por qué crear un idioma?

Crear un lenguaje requiere comprender varias capas al mismo tiempo:

- como sintaxis y leído por humanos;
- cómo entiende un analizador esta sintaxis;
- cómo los tipos ayudan a evitar errores;
- cómo les aparecen los errores a quienes programan;
- cómo define una stdlib la experiencia de uso real;
- cómo ejecuta un tiempo de ejecución decisiones de lenguaje;
- cómo las herramientas, LSP, documentos y paquetes hacen que el lenguaje sea utilizable.

Zenith existe porque este camino enseña más que solo leer sobre estos temas.

## La filosofía del lenguaje

Zenith intenta priorizar:

- código explícito;
- baja ambigüedad;
- mensajes de error legibles;
- documentación breve y directa;
- pequeños ejemplos;
- clara separación entre ausencia y fracaso;
- mutación visible;
- paquetes con contratos predecibles.

El lenguaje debe ser amigable para la lectura.

Esto es especialmente importante para personas con TDAH, dislexia o dificultad para mantener el contexto en archivos largos.

## IA como parte del estudio

Zenith también es un estudio sobre cómo la IA puede ayudar en el desarrollo de un lenguaje.

Esto incluye:

- redactar y revisar documentación;
- crear pruebas;
- comparar decisiones de diseño;
- detectar inconsistencias;
- evolucionar herramientas;
- acelerar los ciclos de implementación.

La IA no elimina la responsabilidad técnica. Aumenta la necesidad de especificaciones claras, buenas pruebas y revisiones honestas.

## Madurez

Zenith aún debería tratarse como un lenguaje en evolución.

Algunas partes son experimentales.

Algunas API pueden cambiar.

El repositorio debe dejar claro qué es:

- actual;
- experimentales;
- planeado;
- historia;
- normativa para su implementación.

Esta separación protege el lenguaje de confusión y documentación contradictoria.

## Invitación

Zenith es, ante todo, un proyecto de aprendizaje profundo.

Si a otras personas les resulta útil estudiar compiladores, tiempos de ejecución, sistemas, inteligencia artificial o diseño de lenguajes, eso ya es una victoria.

La atención no se centra en ganarle al mercado.

El objetivo es aprender con rigor, construir con claridad y registrar las decisiones con honestidad.
