# 統合ロードマップ：Zenith Core

このドキュメントは、Zenith言語のコアの技術的統合を追跡し、Ascension Finalに至るまでの道のりを記録します。

## 1. 概要

私は、パフォーマンスや親しみやすい読解可能性を損なうことなく、認知のアクセシビリティ、構文の明快さ、および技術的な成熟度に焦点を当てた言語としてZenithを維持しています。

## 2. フェーズ 1〜3: 心臓部 (v0.2.5) - 完了

フォーカス: 文法、型システム、および基本インフラストラクチャの確立。

- [X] ADT（代数的データ型）と `match`
- [X] `trait`、ジェネリクス、 `where` を備えた型システム
- [X] 統合された async/await
- [X] `native lua` との相互運用性
- [X] ラムダ、スプレッド、スライシング、および表面上の人間工学

## 3. フェーズ 4: ツーリングと基盤 (v0.2.5) - 完了

フォーカス: Zenithが現実世界と対話できるようにする。

- [X] ビルドと実行をサポートする安定したCLI
- [X] `.ztproj` によるプロジェクトシステム
- [X] `std.os`、 `std.fs`、 `std.json` などのベースライブラリ

## 4. フェーズ 5: ユーティリティとエコシステム (v0.2.6 - v0.2.7) - 完了

フォーカス: 実用的な使用のためのツーリングの拡大。

- [X] `std.time` と `std.text`
- [X] 外部依存関係のためのZPM
- [X] インタラクティブなマニュアルとサポートリソース

## 5. フェーズ 6: シールドと品質 (v0.2.8 - v0.2.9) - 完了

フォーカス: セルフホスティングの前にプラットフォームを保護する。

- [X] CLIに統合されたZTest
- [X] 視覚的なレポートとデータのエクスポート
- [X] 並列実行とスイートの分離
- [X] スナップショットテストと広範なグローバルカバレッジ

## 6. フェーズ 7: Zenith Ascension (v0.3.0 - v0.3.5) - 進行中

フォーカス: セルフホスティング、最適化、およびパイプラインの完全な主権。

- [X] **v0.3.0 (主権)**: Ascensionエンジン完成 (FFI、マルチターゲット、バンドル)。
- [X] **v0.3.1 (ZDoc)**: .zdocファイルのサポートとCLIの純粋性の統合。
- [ ] **v0.3.2 (レキサー)**: 純粋なZenithで書き直された字句解析器。
- [ ] **v0.3.3 (パーサー)**: Zenithに移植されたPratt Parserアルゴリズム。
- [ ] **v0.3.4 (バインダー)**: 記号表とネイティブな意味解決。
- [ ] **v0.3.5 (エミッター)**: Zenithで動作するコード生成器 (Codegen)。

---
## 7. Ascension以降も続く研究

これらの項目は、Zenith v0.3.1エコシステムへの移行を記録しています。

- [X] **Zenith Doc-Linking (ZDoc)**: ソースコードを汚さずにプロ文とシンボルを関連付ける `@target` タグを使用した、分離されたドキュメントのための外部 `.zdoc` ファイルのサポート。
- [X] **ネイティブ正規表現モジュール**: リズミカルで読みやすいパターンを構築するためのFluent Builderの実装。
- [X] **インタラクティブマニュアル (ZMan)**: 純粋なZenithへのリファクタリング。ネイティブブロックに依存せずに色付けを行うための正規表現システムの統合。
- [X] **CLI純粋性の拡大**: 中央ツールにおける `native lua` の劇的な削減。ZPM、ZTest、およびZManは現在、主権あるZenithでオーケストレーションされています。

## 8. 次のステップ (超越 v1.0)

Ascensionエンジンが完成し、CLIが純粋化されたことで、フォーカスは視覚的および産業的な基盤に移ります。

- [ ] **Zenith Intersect (UI)**: 宣言的でリアクティブなフレームワーク。
- [ ] **Borealis (ゲーム)**: Raylib上の産業用ゲームエンジン。
- [ ] **Atlas (データ)**: SQLite/Postgres用の型安全なORM。
