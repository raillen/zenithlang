# Zenithマニフェスト：主権と認知のアクセシビリティ

このドキュメントは、Zenith言語のアイデンティティ、技術哲学、および戦略的ビジョンを定義します。Zenithは単なるプログラミングツールではなく、複雑さを明快さに変えるために設計された知的インフラストラクチャです。

## 1. 個人的な使命と受容

Zenithは、コンパイラやツールを深く研究する必要性から生まれました。自分自身のツールと戦うことなしに、一歩一歩を進むことができるように設計されています。ADHDや失読症（ディスレクシア）を持つ人々によって、また彼らのために設計されたプロジェクトとして、Zenithは「明快さ」「予測可能性」「受容性」がプログラミング体験の基本であるというコミットメントを掲げています。

アクセシビリティは細部ではなく、基盤です。Zenithは、環境がノイズを減らし、人間の処理リズムを尊重すれば、神経多様な人々でも野心的で技術的なシステムを構築できることを証明しています。

---

## 2. 中心的なテーゼ：インフラとしての可読性

現代のプログラミング言語の多くは、極端な簡潔さのために明快さを犠牲にしています。Zenithは3つの原則を通じて、このロジックを逆転させます。

*   **低い神経摩擦**: プログラマーは構文を解読するのではなく、問題を解決することにエネルギーを費やすべきです。Zenithは、予測可能な垂直方向の視覚的フローを確保するために、明示的なキーワード（end、match、whereなど）を使用します。
*   **視覚的な安定性**: 数学記号や演算子は、意味的に必要な場所にのみ表示されます。文法は主にテキスト形式で、可読性を維持し、失読症に典型的な「文字のダンス」を避けるように設計されています。
*   **暗黙よりも明示**: 魔法のような挙動は避けられます。型には名前が付けられ、失敗はOptionalやOutcomeを通じて処理され、データコントラクトはwhereを通じて宣言されます。

## 3. 技術の三位一体 (v0.3.1)

Zenithの技術的な基盤は、以下の3つのエンジニアリングの柱に基づいています。

1.  **ネイティブな主権 (Ascension)**: コンパイラはZenithで、Zenithのために書かれています。これにより、外部エンジンからの完全な独立性が確保され、ハードウェアを最大限に活用する最適化が可能になります。
2.  **ゼロコスト抽象化**: 高級な機能（トレイト、ジェネリック列挙型、静的型付け）はコンパイル時に解決されます。生成されるコードは、専門家によって手書きされたコードと同じくらい効率的なコードを出力します。
3.  **デフォルトの安全性**: I/O、ネットワーク、パース操作はOutcomeシステムを使用します。コンパイラは、一般的なランタイムエラーが製品環境に到達するのを防ぎます。

## 4. エコシステムと公式フレームワーク

Zenithエコシステムは、特殊なツールを通じて汎用的なコアを拡張します。

### 視覚領域とインターフェース
*   **Meridian**: グラフィカルユーザーインターフェース（GUI）を構築するための宣言的でリアクティブなフレームワーク。
*   **Borealis**: FFIを介してPrism上に構築された、産業レベルのパフォーマンスに焦点を当てた2Dゲームエンジン。

### データインテリジェンスとウェブ
*   **Atlas**: SQLiteとPostgreSQLをサポートする型安全なデータベースフレームワーク（ORM）。
*   **Intersect**: ブラウザ上でネイティブZenithを可能にする、ウェブ向けMVVMフレームワーク。
*   **Orion**: 高パフォーマンスなAPIとサービスのための非同期HTTPサーバー。

### ツーリングと言語知能
*   **Forge**: エコシステムのためのビジュアルエディタおよびオーサリング環境。
*   **Compass**: コード知能を提供する公式のランゲージサーバー（LSP）。

---

## 5. 配信戦略

Zenithは技術的な摩擦のないソフトウェア配信を保証します。

*   **シードモード (Standalone)**: VMと最適化されたコードを含む単一ファイルの実行ファイルを生成します。
*   **コロニーモード (Bundle)**: すべてのネイティブ依存関係が自動的に管理される、整理されたディレクトリ構造（/dist）を生成します。
