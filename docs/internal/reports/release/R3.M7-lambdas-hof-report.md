# R3.M7 Lambdas and HOF Report

- Date: 2026-04-24
- Status: complete with one unrelated suite failure expected from Borealis
- Scope: expression lambdas, first HOF stdlib subset, formatter guidance, perf probe

## Summary

R3.M7 is implemented.

Zenith now supports expression lambdas using:

```zt
func(value: int) => value * 2
```

This is syntax sugar for closures v1.

It uses the same immutable capture and managed `func(...)` runtime model.

## Delivered

- Lexer token for `=>`.
- Parser support for `func(...) => expr`.
- Type checker infers lambda return from the expected `func(...) -> ...` type.
- HIR lowering carries the inferred closure return type.
- Formatter preserves the canonical expression-lambda form.
- `std.collections` includes:
  - `map_int`
  - `filter_int`
  - `reduce_int`
- Positive behavior fixture covers lambda capture plus HOF chain.
- Negative behavior fixture covers return mismatch diagnostics.
- Perf quick benchmark `micro_lambda_hof_run` was added.
- Legibility guidance was published.
