# Zenith 0.4.1-alpha.1

Pre-release focused on the developer experience: LSP/VSCode, simplified project creation, single-file workflows, and installable packages for Windows and Linux.

This is still an alpha release. Breaking changes may happen before 1.0.

## Highlights

- LSP and VSCode extension brought close to the 1.0 target.
- `zt create --app` now creates the shorter current hello world:

```zt
namespace app.main

func main()
    print("Hello from Zenith")
end
```

- `print`, `read`, `debug`, and `type_name` are treated as builtins by the LSP.
- `print()` is no longer suggested as an unqualified shortcut for `io.print(...)`.
- Single-file mode is available for quick scripts:
  - `zt run file.zt`
  - `zt check file.zt`
  - `zt build file.zt`
  - `zt emit-c file.zt`
- Version reporting is available through `zt --version`, `zt -v`, and `zt version`.
- CLI input handling was improved for missing option values and source-file/project command mismatch.
- Release packages were generated for Windows, Debian/Ubuntu, Fedora/RPM, and Arch-based systems.

## Install

### Windows

Download and run:

- `zenith-0.4.1-alpha.1-windows-amd64-setup.exe`

The installer includes:

- `zt.exe`
- `zpm.exe`
- `zt-lsp.exe`
- `stdlib`
- `runtime`

It can also add Zenith to PATH and configure `ZENITH_HOME`.

### Debian / Ubuntu

```bash
sudo apt install ./zenith-0.4.1-alpha.1-linux-amd64.deb
```

### Fedora / RPM-based

```bash
sudo dnf install ./zenith-0.4.1-alpha.1-linux-x86_64.rpm
```

### Arch Linux

```bash
sudo pacman -U ./zenith-0.4.1-alpha.1-linux-x86_64.pkg.tar.zst
```

## VSCode

If attached to this release, install the extension package:

- `zenith-vscode-1.0.0.vsix`

Command:

```powershell
code --install-extension zenith-vscode-1.0.0.vsix --force
```

## Notes

- Linux packages install Zenith under `/usr/local/lib/zenith`.
- Linux wrappers are exposed under `/usr/local/bin`.
- Linux packages install `/etc/profile.d/zenith.sh` to define `ZENITH_HOME`.
- This release is intended for testing the current language/tooling workflow before a stable 1.0.
