# R4.M4 - Alpha.2 Language Core Closure Report

Date: 2026-04-25
Status: fechado localmente
Scope: language core, local Windows x64 validation

## Decisao de fechamento

Alpha.2 local esta fechado para o core da linguagem.

Fechar aqui nao significa que todos os itens imaginados para R4 foram
implementados. Significa que o corte Alpha.2 tem um contrato coerente,
validado e documentado.

Itens grandes que mudam a linguagem foram movidos para roadmap futuro.
Isso reduz risco de regressao e evita adicionar semantica publica sem RFC.

## O que esta fechado

- build local do compilador;
- build local do ZPM;
- driver com `zt zpm` e `zt pkg`;
- gate raiz da stdlib;
- smoke suite;
- lazy explicit order fixtures;
- FFI 1.0 atual;
- stress test de `Shared<T>` no runtime C;
- compilacao concorrente/hardening;
- documentacao de escopo no checklist e roadmap v4.

## O que nao foi incluido

### Concorrencia de surface

Nao entram neste corte:

- `task` como keyword;
- `chan<T>`;
- API publica de jobs;
- modelo completo de cancelamento;
- propagacao automatica de estado compartilhado.

Motivo:

- isso muda a semantica publica;
- `task` pode conflitar com identificadores existentes;
- channels precisam de desenho proprio de ownership e backpressure.

### FFI avancado

Nao entram neste corte:

- raw pointers;
- C string automatica;
- callbacks;
- calling convention customizavel;
- symbol renaming.

Motivo:

- a spec FFI 1.0 atual bloqueia esses recursos;
- ponteiros crus exigem contrato de lifetime;
- callbacks dependem de suporte completo de callables/delegates.

### Optimizer ARC

Nao entram neste corte:

- inline ARC;
- eliminacao global de retain/release redundante;
- analise de escape dedicada.

Motivo:

- e trabalho de optimizer;
- precisa de benchmarks antes de virar gate;
- nao bloqueia correcao do core Alpha.2.

## Validacao local

Gates usados como evidencia:

- `python build.py`
- `./zt.exe check zenith.ztproj --all --ci`
- `python run_suite.py smoke --no-perf`
- `python run_all_tests.py` (`190 total`, `188 pass`, `0 fail`, `2 skip`)
- `python tests/hardening/test_concurrent_compilation.py`
- runtime C `test_shared_text`
- FFI positive/negative fixtures.

## Proximo ciclo recomendado

A proxima fase deve abrir RFCs separadas para:

1. concorrencia publica (`task`, `chan<T>`, jobs);
2. FFI avancado (raw pointers, callbacks, ABI annotations);
3. optimizer ARC.

Esses temas nao devem ser tratados como pequenos bugs restantes do checklist.
Eles sao mudancas de linguagem.
