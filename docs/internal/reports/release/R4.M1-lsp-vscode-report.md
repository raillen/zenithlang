# R4.M1 - Compass LSP v1 + VSCode beta

Status: implementado como beta local.

Data: 2026-04-25

## Entregas

- Diagnostics em tempo real: implementado em `compiler/driver/lsp.c`.
- Autocomplete: implementado para palavras-chave, tipos, builtins, snippets basicos, simbolos do arquivo aberto, variaveis locais e parametros.
- Autocomplete de builtins: implementado para `fmt`, `len()`, `to_text()`, `check()`, `panic()` e conversoes numericas como `int()`/`float()`.
- Autocomplete da stdlib: implementado para membros publicos de modulos `std.*` importados. Exemplo: `import std.io as io` habilita `io.print`, `io.read_line`, `io.read_all` e atalhos qualificados como `print()` -> `io.print(...)`.
- Autocomplete de imports: implementado para `std.*` e namespaces de módulos `.zt` indexados no workspace.
- Autocomplete contextual de membro: implementado para `list<T>.get`, `map<K,V>.get`, campos de `core.Error` e campos de structs tipadas no arquivo aberto.
- Autocomplete contextual para despacho dinamico: `any<Trait>`/`dyn<Trait>` sugere metodos declarados na trait.
- Autocomplete contextual com inferencia leve: `using`, literais e chamadas resolvidas ajudam a sugerir membros sem tipo explicito.
- Indice de workspace: a extensao indexa arquivos `.zt` do workspace e sincroniza create/change/delete.
- Indice import-aware: completion/definition cross-file consideram namespace, `import ... as ...` e `public`.
- Hover de simbolos: implementado para simbolos top-level indexados e simbolos locais do arquivo aberto.
- Go-to-definition: implementado para simbolos top-level indexados, simbolos locais do arquivo aberto e outro arquivo.
- References: implementado para simbolos top-level indexados, simbolos locais e usos via `alias.membro`.
- Rename: implementado para simbolos top-level indexados e simbolos locais.
- Signature Help: implementado para chamadas de funcoes top-level indexadas.
- Semantic Tokens: implementado sobre o lexer oficial para highlight semantico leve.
- Document Symbols/Outline: implementado para funcoes, structs, traits, enums, consts, vars e filhos diretos como fields/metodos/variants.
- Workspace Symbols: implementado sobre documentos `.zt` indexados no workspace.
- Formatacao: implementada via formatter oficial do AST.
- Comandos `check/build/run`: implementados na extensao VSCode beta.
- Activation event manual removido: VSCode ja ativa a extensao a partir das contribuicoes de linguagem.

## Risco residual

- O indice atual e textual/in-memory sobre arquivos `.zt` sincronizados.
- A resolucao de namespace/import/public cobre os fluxos de editor, mas nao substitui o typechecker completo do projeto.
- References ainda e textual sobre documentos indexados, mas cobre simbolos locais com escopo simples no arquivo aberto.
- Rename ainda e conservador: cobre simbolos top-level indexados e simbolos locais, evitando strings/comentarios simples e homonimos fora do escopo.
- Signature Help cobre chamadas diretas, `alias.func(...)` e metodos via `apply`; sobrecarga futura ainda nao esta coberta.
- Signature Help cobre builtins de chamada como `len(...)`, `to_text(...)`, `check(...)`, `panic(...)` e conversoes numericas; `fmt` e uma expressao especial sem parenteses.
- Semantic Tokens usa o lexer oficial para a sintaxe atual; ainda nao substitui analise semantica completa do typechecker.
- Completion de membro para campos de struct e metodos via `apply` usa documentos indexados; casos genericos complexos ainda dependem do typechecker.
- Completion de membro, Hover e Signature Help ja consultam a API interna do checker para resolver tipos explicitos quando o documento fornece informacao suficiente. Se a resolucao vier parcial, por exemplo `unknown`, o LSP usa o fallback leve anterior.
- O fallback leve permanece ativo para arquivo incompleto durante digitacao, mantendo completion basica mesmo antes do `end`.
- Diagnostics publicados pelo LSP deduplicam entradas identicas de parse/binder/checker para reduzir ruido.
- Completion de membro resolve generics simples de struct. Exemplo: `GenericBox<int>.item` mostra `item: int` quando o campo foi declarado como `T`.
- Completion de enum sugere variantes em `Enum.` e em `case` dentro de `match`.
- Completion de `case` tambem sugere `some`/`none` para `optional<T>` e `success`/`error` para `result<T, E>`, sem misturar sugestoes globais nesse contexto.
- O LSP nao sugere metodos ainda nao aceitos pelo compilador atual, mesmo que estejam planejados ou documentados como direcao de linguagem.
- O LSP usa sincronizacao full-document.
- A extensao ainda e beta local; publicacao exige empacotamento `.vsix` e validacao manual.
- Validacao visual no VSCode precisa ser feita em uma janela Extension Development Host.
