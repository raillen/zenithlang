## Resumo

R4.M0 estabilizou a base da linguagem para Alpha.2 local.

O foco foi remover regressions que impediam os gates principais e validar o
caminho de runtime mais sensivel deste corte: ARC/Shared sob carga.

Resultado pratico:

- build principal voltou a passar;
- `zt.exe` e `zpm.exe` sao gerados localmente;
- projeto raiz passa em `zt check --all --ci`;
- smoke suite passa;
- suite completa passa com 190 total, 188 pass, 0 fail, 2 skip;
- `Shared<T>` tem stress test concorrente no runtime C;
- FFI 1.0 foi revalidado nos casos positivos e negativo atuais.

## Entregas

### 1. Build e driver

- `python build.py` gera `zt.exe`.
- `python build.py` gera `zpm.exe`.
- `zt pkg` e `zt zpm` sao aceitos pelo driver.
- O standalone `zpm.exe` reutiliza o driver sem duplicar estado global.

Arquivos principais:

- `compiler/driver/main.c`
- `compiler/driver/zpm_main.c`
- `compiler/driver/zpm.c`
- `compiler/driver/zpm.h`
- `compiler/driver/driver_internal.h`
- `compiler/driver/paths.c`

### 2. Gate raiz da stdlib

Correcoes aplicadas:

- `lazy<T>` registrado como tipo builtin aceito pelo binder.
- Chamadas qualificadas privadas dentro do mesmo namespace foram permitidas.
- Isso evita tratar `fs.map_host_error` e chamadas internas similares como acesso externo indevido.

Arquivos principais:

- `compiler/semantic/binder/binder.c`
- `compiler/semantic/types/checker.c`

### 3. Runtime Shared/ARC sob carga

Foi adicionado stress test concorrente para `Shared<T>`/texto no runtime C.

O teste cria varios workers nativos e repete operacoes de retain, borrow,
snapshot e release. No final, o refcount precisa voltar para 1.

Arquivo principal:

- `tests/runtime/c/test_shared_text.c`

Resultado esperado:

```text
Runtime shared ref tests OK
```

### 4. FFI 1.0 revalidado

A spec atual continua sendo `language/spec/ffi.md`.

Validacoes usadas:

- `tests/behavior/extern_c_text_len_e2e`
- `tests/behavior/extern_c_puts_e2e`
- `tests/behavior/extern_c_struct_arg_error`

Decisao mantida:

- raw pointers, callbacks e ABI annotations nao fazem parte deste corte.

## Limites conhecidos

Nao fazem parte de Alpha.2 local:

- keyword `task` oficial;
- `chan<T>`;
- raw pointers na surface FFI;
- conversao automatica `text`/`string` para C string;
- calling convention customizavel;
- callbacks C chamando Zenith;
- optimizer ARC dedicado.

Esses itens estao registrados no roadmap v4 como futuro trabalho de linguagem.
