# R3.M2 Residual Scope Report

Status: residual scope formally deferred out of R3
Date: 2026-04-24
Owner: R3.M2 follow-up

## Summary

R3.M2 delivered the Phase 1 concurrency base.

R3.M2 did not deliver the full original milestone scope.

Decision 091 formally moves the remaining scope out of R3.

R3 may claim:

- R3.M2 Phase 1 complete;
- full concurrency surface deferred to `R4.CF1`.

R3 must not claim:

- `task` is implemented;
- `channel` is implemented;
- `Shared<T>` is implemented;
- runtime race/cancellation behavior is shipped.

## Delivered In R3.M2

- authoritative concurrency spec;
- boundary-copy contract;
- `std.concurrent.copy_*` helpers;
- deterministic copy behavior;
- namespace-state rules for `public var` under the current single-owner model.

## Moved Out Of R3

| Item | R3 Status | New Destination |
|---|---|---|
| official `task` + `channel` model | not shipped | `R4.CF1` |
| runtime tests for race/order/cancellation | order via copy covered; race/cancellation not shipped | `R4.CF1` |
| explicit `Shared<T>` path | not shipped | `R4.CF1` |
| checker-level `transferable` predicate | partial groundwork only | `R4.CF1` |

## Release Impact

R3.M9 is no longer blocked by this residual scope.

R3.M9 remains blocked by the release gates documented in the release report.

## Recommendation

Use `R4.CF1` as the planning home for the moved work.

Reason:

- `task`;
- `channel`;
- cancellation;
- `Shared<T>`;
- and race tests

are not small polish items. They change runtime semantics and need their own design/review loop.

## Required Before Final R3 Publication

- do not claim full runtime concurrency;
- mention that R3 shipped Phase 1 only;
- keep final publication gated by PR/nightly release gates.
