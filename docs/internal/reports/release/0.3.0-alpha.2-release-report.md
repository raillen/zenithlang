## 3) Key Fixes (by PLI)

- `PLI-01`, `PLI-08`: `ZT_LEX_MAX_STRING_LEN = 1 MiB` + `lexer.token_too_long` / `lexer.unterminated_string`.
- `PLI-02`, `PLI-09`: emitter symbol-collision guard and part-buffer bump from `64` to `256` bytes.
- `PLI-03`, `PLI-05`, `PLI-06`: batched closure (details in the PLI report).
- `PLI-04`: formatter idempotence runner + `pr_gate` tooling integration.
- `PLI-07`: use-after-free on `source_text` in `compiler/driver/project.c` removed.
- `PLI-10`: six distinct formatter/parser bugs (string quoting, type-parameter delimiters, `match`/`case` `end`, enum variant commas, `where` emission, comment stealing). All 9 formatter cases converge.

## 4) Artifact Policy

No new Windows installer artifact was cut for `0.3.0-alpha.2`.

Rationale:

- the public alpha contract was already published as `zenith-0.3.0-alpha.1-windows-amd64.zip` in `R2.M12`;
- `0.3.0-alpha.2` is a source-tree and docs cut (no user-visible feature), so we publish the CHANGELOG, notes and compatibility record, but keep the previously published binaries as the current alpha install path;
- the next published binary bundle will be emitted when a new feature cut or a binary-impacting fix requires it.

Build inputs that still reference `0.3.0-alpha.1` and remain intentionally un-bumped for this cut:

- `installer/zenith.iss` (`MyAppVersion`)
- `tools/build_installer.ps1` (`-Version` default)
- `tools/build_linux_packages.py` (`DEFAULT_VERSION`)

These will be bumped together with the next published binary artifact, not here.
