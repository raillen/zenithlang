# 0.3.0-alpha.2 Compatibility Record

Date: 2026-04-23

## Compatibility Statement

- Release target confirmed as `0.3.0-alpha.2`.
- Pre-`1.0.0` compatibility policy remains active (breaking changes allowed between pre-releases).
- This cut carries no intentional breaking surface change vs `0.3.0-alpha.1`. Any observable output difference is a correctness fix backed by a new or existing gate.

## Source Changes With User-Visible Effect

- `zt fmt` is now guaranteed idempotent on canonical cases; files that previously survived formatting by accident (via the comment-stealing path in `compiler/frontend/parser/parser.c`) may produce a different, canonical result now. The new output is stable under repeated formatting and is backed by `tests/formatter/run_formatter_idempotence.py`.
- `zt check` / `zt build` now fail fast with an explicit diagnostic when two Zenith symbols would map to the same mangled C name, instead of emitting colliding C symbols. Previously-compiling projects that relied on such a collision (none known in stdlib/tests) would now get a clean diagnostic.

## platform/ Status

- `stdlib/platform/` stays internal and placeholder-only — unchanged from `0.3.0-alpha.1`.
- Public API status: not exposed for app consumption.

Evidence:

- `stdlib/platform/README.md` documents internal-only contract.
- `stdlib/platform/.gitkeep` keeps the folder structure only.

## Artifact Compatibility

- No new Windows binary artifact is published for `0.3.0-alpha.2`. The currently published alpha install path remains `zenith-0.3.0-alpha.1-windows-amd64.zip` / `.checksums.txt` from `R2.M12`.
- Consumers pinning to `0.3.0-alpha.1` are unaffected. Consumers building from source at this tag get the PLI-01..PLI-10 fixes.

## Residual Risk

- Any future expansion of `stdlib/platform/` must still preserve:
  - no overlap with `std.os`
  - no overlap with `std.os.process`
- Runtime thread-safety boundary and RC cycles remain explicitly tracked as `Risk` in `language/surface-implementation-status.md`.
