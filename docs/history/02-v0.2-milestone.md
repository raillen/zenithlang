# Marco Histórico: Zenith Core v0.2

O fechamento do **Zenith Core v0.2** marcou a transição do projeto de um protótipo sintático para uma linguagem com semântica própria, lowering e base de stdlib reutilizável.

## O que o v0.2 consolidou

### 1. Lowering e IR

O compilador ganhou uma etapa intermediária responsável por:

- desestruturação
- lowering de `match`
- contratos `where`
- otimizações como constant folding

### 2. Modelagem mais rica

O v0.2 estabeleceu os pilares de modelagem atuais:

- `type`
- `union`
- `Optional`
- `Outcome`
- `trait`
- `apply Trait to Struct`
- genéricos com `where T is Trait`

### 3. Sintaxe moderna do Zenith

Foi nessa fase que se consolidaram recursos que ainda definem a linguagem hoje:

- `@campo` para acesso a `self`
- `match` com padrões de lista, struct e variantes
- `check`
- `attempt/rescue`
- spread e slicing com `..`
- funções `async` e `await`

## Impacto no projeto

O v0.2 virou a base sobre a qual o repositório atual continua evoluindo. As expansões posteriores de stdlib, como `std.os`, `std.fs`, `std.json`, `std.time` e o novo `std.os.process`, só foram possíveis porque esse marco estabilizou:

- parser
- binder
- lowering
- codegen Lua
- runtime base de `Optional`, `Outcome`, listas, mapas e async

## O que veio depois

Depois do core v0.2, o foco mudou de “provar a linguagem” para “torná-la utilizável no dia a dia”:

- fundação de stdlib
- interoperabilidade com sistema operacional
- manipulação de arquivos e JSON
- documentação pedagógica
- estabilização do pipeline async

## Status

Marco concluído. O repositório atual já está em uma fase posterior de expansão de stdlib e acabamento de runtime.
