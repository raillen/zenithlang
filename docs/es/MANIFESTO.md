# Manifiesto Zenith: Soberanía y Accesibilidad Cognitiva

Este documento define la identidad, la filosofía técnica y la visión estratégica del lenguaje Zenith. Zenith no es solo una herramienta de programación; es una infraestructura cognitiva diseñada para transformar la complejidad en claridad.

## 1. Misión Personal e Inclusión

Zenith nació de la necesidad de estudiar compiladores y herramientas en profundidad, sin convertir cada paso en una lucha contra la propia herramienta. Como un proyecto diseñado por y para personas con TDAH y dislexia, Zenith asume el compromiso de que la claridad, la previsibilidad y la acogida son fundamentales para la experiencia de programar.

La accesibilidad no es un detalle; es la base. Zenith demuestra que las mentes neurodivergentes pueden construir sistemas técnicos ambiciosos cuando el entorno reduce el ruido y respeta el ritmo de procesamiento humano.

---

## 2. Tesis Central: La Legibilidad como Infraestructura

La mayoría de los lenguajes modernos sacrifican la claridad en favor de una concisión extrema. Zenith invierte esta lógica a través de tres principios:

*   **Bajo Fricción Neural**: El programador debe gastar energía resolviendo el problema, no descifrando la sintaxe. Zenith utiliza palabras clave explícitas (como end, match, where) para garantizar un flujo visual vertical y previsible.
*   **Estabilidad Visual**: Los símbolos matemáticos y operadores aparecen solo donde son semánticamente necesarios. La gramática permanece predominantemente textual y legible para evitar el "baile de letras" típico de la dislexia.
*   **Explícito sobre Implícito**: Se evitan los comportamientos mágicos. Los tipos son nombrados, los fallos se manejan vía Optional y Outcome, y los contratos de datos se declaran vía where.

## 3. La Trinidad Técnica (v0.3.1)

La fundación técnica de Zenith descansa sobre tres pilares de ingeniería:

1.  **Soberanía Nativa (Ascension)**: El compilador está escrito en Zenith para Zenith. Esto garantiza una independencia total de motores externos y permite optimizaciones que explotan al máximo el hardware.
2.  **Abstracciones de Costo Cero**: Las funcionalidades de alto nivel (Traits, Enums Genéricos, Tipado Estático) se resuelven en tiempo de compilación. El código generado es tan eficiente como el código escrito a mano por expertos.
3.  **Seguridad por Defecto**: Las operaciones de E/S, red y parsing utilizan el sistema Outcome. El compilador impide que los fallos comunes de tiempo de ejecución lleguen a producción.

## 4. Ecosistema y Frameworks Oficiales

El ecosistema Zenith expande el núcleo genérico a través de herramientas especializadas:

### Dominio Visual e Interfaces
*   **Meridian**: Framework declarativo y reactivo para la construcción de interfaces gráficas de usuario (GUI).
*   **Borealis**: Motor de juegos 2D enfocado en el rendimiento industrial, construido sobre Prism vía FFI.

### Inteligencia de Datos y Web
*   **Atlas**: Framework de base de datos de tipo seguro (ORM) con soporte para SQLite y PostgreSQL.
*   **Intersect**: Framework web MVVM para el navegador, permitiendo Zenith nativo en el browser.
*   **Orion**: Servidor HTTP asíncrono para APIs y servicios de alto rendimiento.

### Herramientas e Inteligencia
*   **Forge**: Editor visual y entorno de autoría para el ecosistema.
*   **Compass**: Servidor de lenguaje oficial (LSP) que proporciona inteligencia de código.

## 5. Estrategia de Distribución

Zenith garantiza la entrega de software sin fricción técnica:

*   **Modo Semilla (Standalone)**: Genera un ejecutable de archivo único que contiene la VM y el código optimizado.
*   **Modo Colonia (Bundle)**: Genera una estructura de directorios organizada (/dist) con todas las dependencias nativas gestionadas automáticamente.
