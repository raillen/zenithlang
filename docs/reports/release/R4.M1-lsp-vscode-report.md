# R4.M1 - Compass LSP v1 + VSCode beta

Status: implementado como beta local.

Data: 2026-04-25

## Entregas

- Diagnostics em tempo real: implementado em `compiler/driver/lsp.c`.
- Autocomplete: implementado para palavras-chave, tipos, snippets basicos, simbolos do arquivo aberto, variaveis locais e parametros.
- Autocomplete de imports: implementado para `std.*` e namespaces de módulos `.zt` indexados no workspace.
- Autocomplete contextual de membro: implementado para `list<T>.get`, `map<K,V>.get`, campos de `core.Error` e campos de structs tipadas no arquivo aberto.
- Indice de workspace: a extensao indexa arquivos `.zt` do workspace e sincroniza create/change/delete.
- Indice import-aware: completion/definition cross-file consideram namespace, `import ... as ...` e `public`.
- Hover de simbolos: implementado para simbolos top-level indexados.
- Go-to-definition: implementado para simbolos top-level indexados, incluindo outro arquivo.
- Formatacao: implementada via formatter oficial do AST.
- Comandos `check/build/run`: implementados na extensao VSCode beta.

## Risco residual

- O indice atual e textual/in-memory sobre arquivos `.zt` sincronizados.
- A resolucao de namespace/import/public cobre os fluxos de editor, mas nao substitui o typechecker completo do projeto.
- Completion de membro para campos de struct e metodos via `apply` usa documentos indexados; casos genericos complexos ainda dependem do typechecker.
- O LSP nao sugere metodos ainda nao aceitos pelo compilador atual, mesmo que estejam planejados ou documentados como direcao de linguagem.
- O LSP usa sincronizacao full-document.
- A extensao ainda e beta local; publicacao fica para `R4.M5`.
- Validacao visual no VSCode precisa ser feita em uma janela Extension Development Host.
