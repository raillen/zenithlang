# R3.M2 - Concurrency Base Report (Phase 1)

- Date: 2026-04-23
- Scope: M35.1 concurrency base
- Status: delivered (Phase 1 complete) - Phases 3-5 explicitly deferred
- Upstream: `docs/planning/roadmap-v3.md`, `docs/planning/checklist-v3.md`, `language/decisions/087-concurrency-workers-and-transfer-boundaries.md`
- Downstream: `docs/reports/R3-risk-matrix.md`

## 1) Framing

The `R3.M2` roadmap item originally enumerated `task`, `channel`, `Shared<T>`, race/order/cancellation tests, and a full concurrency spec. Decision 087 (accepted 2026-04-23) formalized an explicit 5-phase rollout:

- Phase 1: boundary contract + typed copy helpers
- Phase 2: checker `transferable` analysis
- Phase 3: `jobs.spawn/join`
- Phase 4: move-based transfer + `channels`
- Phase 5: `Shared<T>`, `atomic<T>`

`R3.M2` ships **Phase 1** with supporting spec and integration with `R3.P1.A`. Phases 3-5 are explicitly deferred and tracked as risks `R3-RISK-010`, `R3-RISK-011`, `R3-RISK-012`.

This matches the precedent set by `R3.B7` in the Borealis trilha, which closed as a partial delivery with clear scope.

## 2) Delivered

### 2.1 Boundary Contract (Phase 1)

- `stdlib/std/concurrent.zt` exposes 8 typed copy helpers:
  - `copy_int`, `copy_bool`, `copy_float`, `copy_text`, `copy_bytes`, `copy_list_int`, `copy_list_text`, `copy_map_text_text`.
- Runtime implementation backs text/bytes/list/map helpers via `extern c` calls into `runtime/c/zenith_rt.c` (`zt_thread_boundary_copy_text`, `zt_thread_boundary_copy_bytes`, `zt_thread_boundary_copy_list_i64`, `zt_thread_boundary_copy_list_text`, `zt_thread_boundary_copy_map_text_text`).
- Copy semantics: deep copy, source not mutated by copy mutation, repeated copy calls produce observationally equal outputs.

### 2.2 Authoritative Spec

- `language/spec/concurrency.md` (new): single authoritative document for the Zenith concurrency model.
  - Default execution model (single isolate, non-atomic ARC).
  - Boundary contract (copy / move / shared wrapper).
  - Transferable shapes and non-transferable classes.
  - Determinism rule (observational equality for copy, source isolation for list/map).
  - Namespace state under concurrency (R3.P1.A integration).
  - Implementation phases table (Phase 1 delivered, Phases 2-5 mapped).
  - Tests inventory (positive, negative, determinism).

### 2.3 R3.P1.A Integration

- `language/decisions/087-concurrency-workers-and-transfer-boundaries.md` now includes section `Namespace State And Concurrency (R3.P1.A Integration)`.
- Normative rules integrated:
  - `public var` is namespace-owned;
  - default model is single-owner under concurrent execution;
  - no implicit promotion to shared/atomic wrapper;
  - crossing a worker/task boundary uses the copy contract;
  - shared-mutation requires explicit Phase 4/5 surfaces.
- Cross-reference: `docs/reports/R3.P1.A-namespace-shared-state-analysis.md`.

### 2.4 Tests

Positive (pre-existing):

- `tests/behavior/std_concurrent_boundary_copy_basic`
  - exercises all 8 `copy_*` helpers with representative values and verifies post-copy equality.

Negative (pre-existing):

- `tests/behavior/std_concurrent_boundary_copy_unsupported_error`
  - rejects a non-transferable struct passed to `copy_text`; diagnostic fixture in `tests/fixtures/diagnostics/std_concurrent_boundary_copy_unsupported_error.contains.txt`.

Determinism (new, this milestone):

- `tests/behavior/std_concurrent_boundary_copy_determinism`
  - exercises repeated `copy_int`/`copy_text`/`copy_list_int` calls for observational equality;
  - verifies boundary isolation (mutating the copy of a `list<int>` does not mutate the source);
  - verifies order invariance (same sequence of copy calls with same inputs yields same intermediate values across repeated runs).
  - run result: `verification ok` on `zt check`; `exit code: 0` on `zt run`.

## 5) Acceptance Statement

Against the `R3.M2` acceptance criterion ("concurrency base executavel sem violar isolamento/ownership"):

- **Met for Phase 1**: copy-boundary contract executes and preserves isolation. Source is not mutated by copy mutation. Determinism is verified. Ownership model (single isolate, `public var` single-owner) is explicitly documented.
- **Deferred**: runtime execution of `task`/`channel` workloads is not part of this cut.

`R3.M2` closes as **Phase 1 complete, Phases 3-5 tracked**, consistent with Decision 087.

## 6) Residual Risk

- Phase 2 (`transferable` predicate and unified boundary diagnostic) is partial in the checker; needs surface consolidation before Phase 3.
- Copy helpers cover only a curated set of shapes; additional shapes (for example `list<struct>`) are not yet covered.
- Implementation phases 3-5 remain significant engineering effort; the risk matrix reflects their scope and deadlines.
