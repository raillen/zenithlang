# R3.M9 Known Limits And Risk

Status: published for release candidate
Date: 2026-04-24

## Short Version

The R3 candidate is useful.

It is not publishable yet.

The package installs and runs, but the release gates are red.

## Known Limits

- `public var` is namespace state, not a global variable system.
- `public var` writes remain restricted to the owner namespace.
- concurrency has boundary-copy helpers, but not full `task`/`channel` runtime yet.
- FFI 1.0 does not yet include callback/delegate interop.
- `dyn Trait` is a minimum subset.
- closures do not support mutable capture yet.
- lambdas are expression-only.
- HOF helpers are not fully generic yet.
- lazy runtime support is currently `lazy<int>` one-shot.

## Release Blockers

- `tests/perf/gate_pr.ps1` failed.
- `tests/perf/gate_nightly.ps1` failed.
- `behavior/borealis_backend_fallback_stub` still needs closure or formal acceptance.
- `hardening/differential_validate_between` needs investigation.
- perf baseline drift needs either optimization or an approved rebaseline.
- R3.M2 residual scope still needs implementation or formal deferral:
  - `task`;
  - `channel`;
  - `Shared<T>`;
  - race/cancellation runtime tests.

## Risk Level

Release risk: high enough to block publication.

Development risk: acceptable for continued iteration.

## Recommended Next Step

Treat this as a release candidate.

Do not tag or announce it as the final R3 release until the gates pass.

Also do not claim full R3.M2 completion until the residual scope is resolved.
