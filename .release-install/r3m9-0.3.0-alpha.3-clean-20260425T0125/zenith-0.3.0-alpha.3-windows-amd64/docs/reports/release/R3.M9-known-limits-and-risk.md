# R3.M9 Known Limits And Risk

Status: published for final R3 release
Date: 2026-04-24

## Short Version

The R3 release is finalized locally with verifiable artifacts.

The package installs and runs.

The release gates exit successfully.

The nightly perf gate uses the documented R3.M9 baseline override.

## Known Limits

- `public var` is namespace state, not a global variable system.
- `public var` writes remain restricted to the owner namespace.
- concurrency has boundary-copy helpers, but not full `task`/`channel` runtime yet.
- FFI 1.0 does not yet include callback/delegate interop.
- `dyn Trait` is a minimum subset.
- closures do not support mutable capture yet.
- lambdas are expression-only.
- HOF helpers are not fully generic yet.
- lazy runtime support is currently `lazy<int>` one-shot.

## Accepted Release Risks

- perf baseline drift is accepted through `docs/governance/perf-overrides/R3.M9-release-baseline-acceptance.md`;
- R3.M2 full concurrency runtime is not shipped in R3. It was formally moved to `R4.CF1`:
  - `task`;
  - `channel`;
  - `Shared<T>`;
  - race/cancellation runtime tests.

## Risk Level

Release risk: accepted for alpha.

Development risk: acceptable for continued iteration.

## Recommended Next Step

Treat this as the finalized local R3 artifact release.

Also do not claim full runtime concurrency in R3. Claim only Phase 1 concurrency base.
