# Zenith 0.3.0-alpha.3-rc.1 Compatibility Report

Status: release candidate compatibility snapshot
Date: 2026-04-24

## Compatibility Decision

This candidate is compatible enough for local validation.

It is not approved as the final public release.

## Expected Compatibility

- Existing `public const` behavior is preserved.
- `public var` is limited to namespace top-level declarations.
- External code can read `public var` through qualified access.
- External code cannot write another namespace's `public var`.
- There is still no canonical `global` variable model.
- Closures capture immutable values in v1.
- Lambdas are expression-only in v1.
- `lazy<int>` is explicit and one-shot.

## Source Compatibility Notes

Some R3 features are intentionally restricted.

This is expected:

- `lazy<T>` exists in the type system, but only `lazy<int>` has runtime helpers now.
- closure mutable capture is rejected;
- HOF helpers are integer-first, not generic-first;
- FFI callbacks remain future work;
- concurrency `task`/`channel` remain future phases.

## Runtime Compatibility Notes

The runtime now includes:

- closure fat pointers;
- dynamic dispatch support structures;
- `lazy<int>` support;
- runtime shielding paths used by FFI.

These changes increase runtime surface area.

That is why the perf gate must be green before final release.

## Install Compatibility

Clean install from the candidate package passed.

Evidence:

- `docs/reports/release/artifacts/zenith-0.3.0-alpha.3-rc.1-clean-install.log`

## Known Compatibility Blockers

- `behavior/borealis_backend_fallback_stub` fails in PR gate.
- `hardening/differential_validate_between` fails in PR gate.
- perf release-gate baseline comparisons fail.

## Final Compatibility Status

Candidate compatibility: pass for local validation.

Release compatibility: blocked until gate failures are resolved.
